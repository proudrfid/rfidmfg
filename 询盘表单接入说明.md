# 询盘表单接入说明

让网站表单提交后,询盘**自动送达**(不再依赖访客的邮箱客户端)。两条路任选其一。表单已写好,你只需把一个网址填进 `script.js` 顶部的 `FORM_ENDPOINT`。

> 现在 `FORM_ENDPOINT = ''`(留空 = 兜底走 mailto)。填上网址即生效。

---

## 方式 A:Formspree(最简单,5 分钟,无需服务器)

适合先快速跑起来,询盘会发到你邮箱并在后台存档。

1. 打开 https://formspree.io ,注册并 **New Form**,收件邮箱填 `vivian@mind.com.cn`。
2. 复制它给的表单地址,形如 `https://formspree.io/f/abcdwxyz`。
3. 打开 `script.js`,把那行改成:
   ```js
   var FORM_ENDPOINT = 'https://formspree.io/f/abcdwxyz';
   ```
4. 保存、重新部署即可。访客提交后你会收到邮件,Formspree 后台也能看到所有线索。
5. (可选)以后想让线索也进飞书,用 Make/Zapier 把 Formspree → 飞书多维表格连起来。

---

## 方式 B:直接进飞书多维表格(用你自己的应用,线索自动入库)

适合想把询盘直接沉淀到飞书 CRM。会用到一个轻量中转(Cloudflare Worker,免费),因为静态网页不能直接写飞书。代码已给你:`feishu-lead-worker.js`。

**第 1 步:建一张线索表**
在你的多维表格里新建数据表「**网站询盘**」,字段:
`姓名`(文本)、`邮箱`(文本)、`产品`(文本)、`留言`(文本)、`来源`(文本)、`提交时间`(日期)。
记下它的 `table_id`(链接里 `?table=` 后那串)和多维表格的 `app_token`(链接 `/base/` 后那串)。

**第 2 步:部署 Worker**
1. 登录 https://dash.cloudflare.com → **Workers & Pages** → **Create** → **Create Worker**。
2. 把 `feishu-lead-worker.js` 的内容整段粘进去,**Deploy**。
3. 进该 Worker 的 **Settings → Variables**,加这几个环境变量:
   - `FEISHU_APP_ID` = 你的 App ID(`cli_a9fadb91b138dcc0`)
   - `FEISHU_APP_SECRET` = 你的 App Secret(**建议先在开放平台重置一次再填**)
   - `APP_TOKEN` = 多维表格 app_token
   - `TABLE_ID` = 「网站询盘」表的 table_id
   - `ALLOW_ORIGIN` = 你的网站地址(如 `https://www.mindrfid.com`)
4. 复制 Worker 网址(形如 `https://xxx.workers.dev`)。

**第 3 步:填进网站**
打开 `script.js`:
```js
var FORM_ENDPOINT = 'https://xxx.workers.dev';
```
保存、部署。提交一次测试,飞书表里应出现新记录。

**前提**:你的应用已开通「查看、评论、编辑和管理多维表格」(`bitable:app`,含写入),并已把这张多维表格加入应用可访问范围——这两步我们之前已经做过。若写入报权限错,回开放平台确认数据范围已包含该表。

---

## 安全提醒

App Secret 之前在对话里以明文出现过,**强烈建议**到飞书开放平台 →「凭证与基础信息」**重置 App Secret**,然后更新 Worker 里的 `FEISHU_APP_SECRET`。Secret 只放在 Worker 环境变量里,不要写进前端 `script.js`。
